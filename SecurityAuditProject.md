# 🔐 Security Audit Report – Botium Toy
- from Google Security Course.
## 📌 Objective
Conduct a security audit of Botium Toys to evaluate existing controls, identify risks, and assess compliance with security best practices and regulatory standards.

---

## 📂 Scope
The audit focused on:

- Internal IT systems and data handling practices  
- Access control mechanisms  
- Data protection and encryption practices  
- Network and endpoint security controls  
- Business continuity and disaster recovery readiness  

---

## ⚠️ Findings
The audit identified several critical security gaps:

- Lack of proper **access control mechanisms**, with all employees having access to sensitive customer data (including PII and payment information)  
- Absence of **encryption** for sensitive data stored locally  
- No implementation of an **Intrusion Detection System (IDS)**  
- Weak and outdated **password policy**, without centralized enforcement  
- No **backup strategy** or **disaster recovery plan**  
- Inadequate **asset management** and lack of visibility over critical systems  

These issues expose the organization to significant risks, including:

- Data breaches  
- Regulatory penalties  
- Operational disruptions  

---

## 🛡️ Existing Controls
The organization has implemented some baseline security measures:

- Configured **firewall** for network traffic filtering  
- Installed and monitored **antivirus software**  
- Ensured **data integrity and availability controls**  
- Established **physical security controls**, including CCTV, locks, and fire detection systems  
- Implemented **privacy policies** and breach notification procedures  

---

## 🚀 Recommendations
To mitigate identified risks, the following actions are recommended:

- Implement the **Principle of Least Privilege (PoLP)** to restrict access to sensitive data  
- Deploy an **Intrusion Detection System (IDS)** for real-time threat monitoring  
- Apply **encryption** to sensitive data at rest and in transit  
- Strengthen **password policies** and implement a centralized password management system  
- Develop and maintain a **backup and disaster recovery plan (DRP)**  
- Improve **asset management** and establish a structured inventory of systems and data  

---

## 📊 Conclusion
Botium Toys currently lacks several critical security controls, placing sensitive data and business operations at risk.

Immediate action is required to strengthen:

- Access control  
- Data protection  
- Monitoring capabilities  

This will ensure alignment with industry best practices and compliance requirements.
